# Future interactive effects — roadmap (2026-06-17)

_Generated by a 9-agent ideate → judge → synthesize panel (5 area specialists → 3 judges: taste/restraint, feasibility/gates, recruiter-impact → 1 synthesis), grounded in the real codebase + CI gates. Every item is buildable; the judges corrected several proposals for content-integrity and gate-safety._

**Gates every item must hold:** the 103-assertion browser self-test (pinned ids/counts/text), axe 0 WCAG A/AA, no horizontal overflow 320–1280px, zero console errors, the ~1.5MB Lighthouse byte budget.

**Weighting:** the index page is already feature-dense, so this tilts toward the **untouched** surfaces — `projects.html` (the case-studies page, which loads no `main.js` today) and the isolated `404.html` — plus the genuinely **under-enhanced** index gaps (the 360-bar status strip, the sparklines, the thesis→DAG seam, the whoami block). It deliberately shows **restraint** on the over-enhanced parts (the DAG, the 15-logo toolchain grid, the contact form that already animates a commit + SHA).

## Do first — the next three (highest leverage, lowest risk)

> **Status:** #1 and #2 ✅ **shipped** 2026-06-17 (see `CHANGES-interactive-2026-06-17.md`). #3 (404 shell) still open.

1. **Compiler-lens CSS glow on `projects.html` hero figures** ✅ SHIPPED _(effort S)_
   Cursor over a case-study hero lights a soft amber phosphor pool that tracks the pointer — quoting the index "compiler lens" so the light page reads as the same product, **without** importing the cursor/WebGL/terminal engine. ~10-line per-figure pointer handler writing `--mx/--my` to a radial-gradient (the CSS-fallback path of the index lens). Zero new DOM, ~0 bytes, no deps, outside the index self-test scope. _Cannot break a gate._

2. **Cross-document deploy-handoff — shared-element View Transition (index card → case study)** ✅ SHIPPED _(effort M)_
   Click a project card on index and its hero image + title physically morph/expand into that case study's hero on `projects.html` — the page "deploys" instead of blink-reloading; back-nav reverses it. Pure CSS cross-document View Transitions (`@view-transition { navigation: auto }` in the shared `main.css` + a **per-slug unique** `view-transition-name` on each card hero+title and its destination `<article>` hero+title). **Zero runtime JS**, so it works on the engine-less page; VT pseudo-elements aren't real DOM, so the pinned 5/15/360 counts are untouched; degrades natively to ordinary navigation. _Guard: per-document name uniqueness (don't also name the `.pj-index` TOC links) — a duplicate active name throws and fails the zero-console gate._

3. **404 "recover" shell — a live micro-REPL that routes you home** _(effort M)_
   A dead URL types a failed-deploy log (`exit code 404 · route not found`), then drops to a live prompt you can type into: `ls` lists the real pages, `cd projects`/`home` navigate, `help`, with a typo-correct hint. A broken link becomes the most in-character moment on the site. Self-contained ~3KB inline script (no `main.js`); `404.html` is outside the self-test runner's page list. _Auto-intro completes <3s and skips on any keypress (WCAG-safe); input labelled, output `aria-live`, unknown input never throws._

## Soon (solid, moderate effort)

- **Cmd-K cross-page case-study navigator** — give `projects.html` a ~2KB inline palette (not `main.js`) and add 5 case-study deep-link entries to the index palette, so Cmd-K is the connective tissue between the two documents. (Palette command array is **not** self-test-pinned — verified.) Reuse the shipped `role=dialog`/focus-trap/Esc pattern.
- **Status-bar inspect-on-hover** — sweep the 360-bar uptime strip; the bar under the cursor lifts/brightens and one shared mono chip reads its status. **Honest states only** (`operational`/`degraded` from the existing color class) — never fabricated per-day percentages. One delegated listener (not 360), one container tab stop + arrow-key readout, `aria-live` chip. Adds zero children to `.sp__bars` (count holds).
- **Sparkline crosshair scrub** — hover a sparkline → a crosshair snaps to the nearest **already-plotted** point with a mono value label (Grafana-style). _Account for `devicePixelRatio` when mapping cursor-x → canvas pixels (same root cause as the dpr-0.5 gotcha); the canvases are `aria-hidden` today, so expose the summary via a sibling, not an `aria-label`._
- **whoami.yaml live uptime field** — after the line-reveal, one value resolves to genuinely-live computed text (`uptime: 11y 4mo` against a **true** career-start epoch — not a third copy of the existing deploy clock). `.pln` **count** is pinned (≥6), not the value text, so a dynamic value is safe.
- **Hidden ops commands in the Cmd-K palette** — `sudo hire`, `rollback` (flips the theme bloom), `status` (jump to the uptime strip), `uptime` (echoes the true value); discovering one unlocks a small dismissible "achievement: root access" toast. Every command routes to a feature that already exists (no dead easter eggs).
- **Contact-form staged-diff mirror (read-only)** — as the recruiter types, a live `git diff --staged` renders beneath the form. **Mirror-only**: it never touches the native Netlify submit path or the required attrs (a hook-gating version breaks self-test 142–147 — see Skip).

## Skip — do NOT build (cram / redundant / would break a gate)

- **DAG rollback branch** — "rollback" already ships 3 ways (konami `body.rollback` is self-test-pinned, the incident game, the theme bloom); a transient DAG recolor races the repeated `#work` sampling → probable false-fail. Effort L, max cram on the densest surface.
- **Scrub-the-deploy (drag an artifact across the pipeline)** — second deterministic motion layer on an already-busy DAG; needs MotionPathPlugin (not vendored) or a packet-path hack; draggable invites overflow/pointer-capture errors.
- **DAG node-inspect terminal card** — the `.dagpop` hover popover (role + tool logos) is already shipped **and pinned**; re-skinning it adds focus/aria complexity for thin value.
- **Status-strip scan-sweep** — generic shimmer on a strip that already gets the scroll-throughput glow; adds no information.
- **Toolchain connection-lines on hover** — the 15-logo grid is the **most-enhanced** surface already (lights each to brand colour); this is the exact "one thing too many" cram tell, and line geometry is brittle on reflow.
- **Incident game with a real wall-clock timer / personal-best** — MTTR `47s` and clock `00:47` are pinned **deterministically**; a real timer fails the gate and a "PB" is meaningless when MTTR is scripted. (The quiet static-postmortem + copy variant, which echoes the scripted 47s, is the stretch item.)
- **Contact-form pre-commit hook-gating** — disabling submit until email-validates rewrites the output strings and breaks self-test 142–147. (The read-only diff mirror is the buildable part.)
- **Grep/filter the whoami.yaml block** — redundant second gadget on the same small block; ship at most one whoami enhancement (the live uptime field).
- **Hero 2.5D parallax + before/after wipe on `projects.html`** — parallax is the canonical "could be any award site" move; the wipe is only honest where a real before/after asset exists, needs a second hero image (byte/CLS pressure) + a keyboard range input, and drags GSAP onto the light page. The compiler-lens glow ties the pages at far lower cost.

## Cross-cutting notes

- **How the two pages tie together** (cheapest first, three reinforcing layers): (1) the compiler-lens CSS glow quotes the index hero on `projects.html` at ~0 cost; (2) the cross-doc View Transition makes a card physically deploy into its case study — the headline connective effect, zero runtime JS; (3) the Cmd-K navigator exposes the 5 case studies as first-class destinations. Together they make the two HTML files read as one product **without** importing `main.js` onto the light page.
- **Accessibility:** every hover/scrub readout needs a non-visual path (`aria-live` chip or visible sibling — the sparkline canvases are `aria-hidden` today, so don't `aria-label` them); the status strip gets **one** container tab stop, never 360 focusable bars (WCAG 1.4.13); the 404 input needs a label + `aria-live` output; the palette keeps `role=dialog` + focus-trap + Esc; any toast is `aria-live` + dismissible, never color-only. Motion stays always-on, but the only auto-playing >5s motion (the 404 intro) completes <3s and skips on keypress; everything else is user-driven or computes once.
- **Content-integrity (non-negotiable for an SRE portfolio):** the status readout shows only honest operational/degraded states (no invented per-day %), the sparkline scrubs only the already-plotted series (no fake ticking feed), whoami uptime maps to a true epoch, and the incident postmortem echoes the scripted 47s. Never invent metrics.
- **Gate-safety rule of thumb** (from the pinned self-test): never add/remove children of `.dnode` (×5), `#toolchain .tool` (×15), or `#statuspage .sp__bars` (360); keep `#about .pln` key/label text byte-identical (values may go dynamic); don't touch the `#commitForm` native submit path or its `/aborting/` + 7-hex-SHA output; treat MTTR `47s`/`00:47` as fixed; for cross-doc VT, assert per-document `view-transition-name` uniqueness. The palette command array, `view-transition-name` attributes, and `404.html` are confirmed **not** pinned — most headroom there.
