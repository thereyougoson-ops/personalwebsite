# Portfolio Review — philiptoulinov.com
_Reviewed 2026-06-17 · live behavioral test + source audit + 5-dimension multi-agent review (performance, SEO, design, content-truth, code/security), each adversarially verified._

## Verdict
This is a **genuinely strong, hand-built site** — top ~5–10% of engineer portfolios. The concept (career rendered as a deploy pipeline, a real interactive shell, a playable on-call incident, a `git commit` contact form) is bespoke and the execution holds up: 0 axe WCAG violations, AA contrast throughout, clean security, a real ~60-assertion self-test suite, and copy that reads like an actual opinionated engineer (zero buzzword filler).

**So the pressing work is almost entirely about getting found, parsed, and verified — not craft.** A recruiter who lands on the page is impressed; the leaks are upstream (ATS, search, social shares, GitHub) and at the "can I verify this?" moment.

The single most damaging issue is invisible to you: **your résumé PDF has no text layer, so applicant-tracking systems parse it as empty and silently filter you out.**

## 🔴 P0 — Fix before you send another application
1. **Résumé PDF is image-only → ATS-invisible.** Confirmed: PyMuPDF + `pdftotext` extract **0 characters** from both pages — it's vector/image art, not text. Greenhouse/Lever/Workday parse it to empty and score it near-zero on keyword match, filtering you out before a human sees it. **Fix:** re-export a text-based PDF from the source doc (or `ocrmypdf in.pdf out.pdf`); verify `pdftotext` returns the full content before replacing `assets/philip-toulinov-resume.pdf`. _(effort: small — highest ROI on the entire site.)_

## 🟡 Performance & polish a perf-literate panel will notice
6. **~2.5s artificial preloader gate.** `runPreloader()` (main.js:255-266) scripts ~2.57s of "deploying…" before content appears. Worse: the instant-path already exists but only triggers on the saved motion-toggle, **not** on `prefers-reduced-motion`. **Best fix:** main.js line ~18 — add `|| reduceQuery.matches` so reduced-motion users skip it; and cut the per-line delays (~340/210ms → ~90ms). A slow-by-design splash undercuts a "fast, reliable delivery" pitch on the metric (LCP) a DevOps panel screenshots. _(trivial)_
7. **Render-blocking 4-family Google Fonts.** index.html:26 loads Bricolage Grotesque + Geist + Geist Mono + JetBrains Mono with wide axes, synchronously, no woff2 preload (3-hop chain) — and it contradicts the README's "no runtime CDN." **Fix:** self-host subset woff2 (you already vendor GSAP/Lenis), `@font-face` + `font-display:swap`, preload the 1–2 above-the-fold faces, drop unused weight ranges. _(medium)_
8. **~900KB of oversized project images.** Six PNGs (66–231KB) served at 1440px natural but displayed ~300px; no `srcset`/WebP/AVIF (projects.html:225+). These screenshots *are* your proof of work. **Fix:** export WebP/AVIF at 640w + 1280w with `srcset/sizes` → ~991KB drops to ~150–250KB. _(small)_
9. **`projects.html` (your best page) is under-optimized for SEO/sharing.** No canonical, no Twitter card, no JSON-LD. **Fix:** mirror index.html's head; add an `ItemList` of `CreativeWork`/`SoftwareSourceCode` for the five builds. _(small)_
10. **Timeline reads stale for a 2026 search.** Last role ends Jan 2024 (index.html:325) but about-copy is present-tense ("These days I own releases end to end") — an unexplained ~2.5yr gap. **Fix:** add a dated "2024–present · independent builds & DevOps consulting" node tying the experience timeline to the projects page. _(small)_
11. **Mobile: floating terminal pill overlaps headings.** `.termdock` (main.css:669) sits at ~62px and crowds the "A career, rendered as a deployment." H1 on phones (see m-work/m-stack). Mobile is where many recruiters open links first. **Fix:** `@media(max-width:760px){.termdock{display:none}}` or dock it bottom-right. _(small)_
12. **No failsafe for the loading lock.** The whole page is `opacity:0` under `.is-loading`, cleared only by `afterLoad()`; `boot()` (main.js:1543) has no try/catch or timeout — one uncaught error = blank page. **Fix:** `setTimeout(()=>document.body.classList.remove('is-loading'),3000)` — literally a deploy health-check timeout, on-brand. _(trivial)_
